## HTTP vs HTTPS

### 1. HTTP (HyperText Transfer Protocol)
* **정의:** 웹 브라우저와 서버 간 데이터를 주고받기 위한 기초 프로토콜.
* **상태 비보존 (Stateless):** 이전 통신 상태를 기억하지 않음 (쿠키/세션으로 보완).
* **비연결성 (Connectionless):** 응답 완료 후 연결 종료.
* **취약점:** 데이터 평문 전송으로 인한 스니핑(도청) 위험 존재.

### 2. HTTPS (HTTP Secure)
* **정의:** HTTP에 SSL/TLS 보안 프로토콜을 추가하여 보안성을 강화한 버전.
* **기능:**
    * **암호화:** 전송 데이터를 암호화하여 기밀성 유지.
    * **데이터 무결성:** 전송 중 변조 여부 확인 가능.
    * **인증:** 접속한 서버가 실제 서버임을 CA(인증기관)를 통해 증명.
* **비용:** 인증서 발급 및 갱신 비용 발생 가능 (최근 무료 인증서 보급 활발).

---

### 3. 주요 차이점 비교

| 구분 | HTTP | HTTPS |
| :--- | :--- | :--- |
| **포트 번호** | 80 | 443 |
| **암호화 여부** | 미적용 | 적용 (SSL/TLS) |
| **보안성** | 낮음 | 높음 |
| **SEO 영향** | 검색 노출 시 상대적 불리 | 검색 노출 시 가산점 부여 |

---

### 4. SSL/TLS Handshake (동작 원리)
1.  **Client Hello:** 브라우저가 지원하는 암호화 방식 전달.
2.  **Server Hello:** 서버가 암호화 방식 선택 및 **공개키가 포함된 인증서** 전달.
3.  **인증서 검증:** 브라우저가 신뢰할 수 있는 CA 리스트를 통해 인증서 유효성 확인.
4.  **대칭키 교환:** 브라우저가 임시 키(Pre-master Secret)를 생성, 서버의 공개키로 암호화하여 전달.
5.  **보안 연결 수립:** 서버가 비밀키로 복호화하여 대칭키 획득 후, 이후 통신 암호화에 사용.
