# OSI 7계층

![OSI_7계층](/01-computer-science-fundamentals/network/img/OSI_7_layers.png)

## OSI 7계층을 나눈 이유

흐름을 한 눈에 알아보기 쉽고 7단계 중 특정한 곳에 이상이 생기면 다른 단계의 장비 및 소프트웨어를 건들이지 않고 이상이 생긴 단계만 고칠 수 있기 때문

## OSI 7계층의 종류

### 7계층- 응용 계층 (Application Layer)

- **역할**
    - 사용자가 직접 눈으로 보고 사용하는 프로그램(UI)과 연결되는 가장 위쪽 계층
- **특징**
    - 웹 브라우저(HTTP), 이메일(SMTP), 파일 전송(FTP) 등 사용자가 네트워크를 직접 사용할 수 있게 해주는 응용 프로그램들이 동작

### 6계층 - 표현 계층 (Presentation Layer)

- **역할**
    - 데이터를 서로 이해할 수 있는 공통된 형식으로 변환하거나, 안전하게 암호화/압축하는 역할
- **3가지 기능**
    1. 송신자에서 온 데이터를 해석하기 위한 응용계층 데이터 부호화, 변화
    2. 수신자에서 데이터의 압축을 풀 수 있는 방식으로 된 데이터 압축
    3. 데이터의 암호화와 복호화
- **대표적인 예시**
    - GIF, JPEG, ASCII

### 5계층 - 세션 계층 (Session Layer)

- **역할**
    - 양쪽 컴퓨터의 프로그램간의 연결(TCP/IP 세션)을 맺고, 유지하고, 끊는 역할
- **특징**
    - 통신이 중간에 끊기면 처음부터 다시 보내지 않음
    - 끊긴 지점부터 다시 복구할 수 있게 도와주는 관리자 역할을 함
    - 응용 프로세스가 통신을 관리하기 위한 방법을 정의

### 4계층 - 전송 계층 (Transport Layer)

- **역할**
    - 양 끝단의 컴퓨터가 데이터를 신뢰성있게 주고받을 수 있게 책임지는 역할
- **특징**
    - 데이터가 순서대로 잘 도착했는지, 중간에 사라지진 않았는지 확인
        - 오류검출 및 복구, 흐름제어와 중복검사
- **전송단위**
    - 세그먼트 (Segment)
- **대표적인 프로토콜**
    - TCP (안전하고 정확), UDP (빠르지만 확인은 생략)

### 3계층 - 네트워크 계층 (Network Layer)

- **역할**
    - 목적지까지 가는 가장 안전하고 빠른 최적의 경로를 찾아주는 내비게이션 역할 (라우팅)
- **특징**
    - IP 주소가 해당 계층에서 쓰임.
    - 사용자의 컴퓨터에서 서버까지 데이터를 어떻게 보낼지 경로를 결정
- **전송단위**
    - 패킷 (Packet)
- **장비**
    - 라우터

### 2계층 - 데이터 링크 계층 (Data Link Layer)

- **역할**
    - 물리적인 선으로 바로 연결된 기기(노드)끼리 데이터를 안전하게 전달하는 역할
- **특징**
    - 1계층 (물리 계층)에서 생긴 오류를 찾아내고 고침.
    - 기기마다 가진 고유한 하드웨어 주소인 **MAC 주소**를 사용
- **전송단위**
    - 프레임 (Frame)
- **장비**
    - 스위치, 브릿지, 이더넷

### 1계층 - 물리 계층 (Physical Layer)

- **역할**
    - 데이터를 전기 신호(0과 1)로 바꿔서 진짜 선으로 쏴주는 역할
- **특징**
    - 랜선, 통신 케이블, 허브 등과 같은 물리적인 장비들이 속함.
    - 데이터가 무슨 내용인지, 에러가 있는지 신경 쓰지 않고 전달만 함.
- **전송단위**
    - 비트 (Bit)
- **장비**
    - 케이블, 허브
