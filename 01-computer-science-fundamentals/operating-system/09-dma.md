## DMA

**Direct Memory Access**

### 1. 개념 및 도입 목적

- **정의:** CPU 개입 없이 입출력(I/O) 장치와 주기억장치(메모리) 간 데이터를 직접 전송하는 기법
- **도입 목적:**
    - 기존 제어 방식의 CPU 자원 낭비 문제 해결
    - 속도가 느린 I/O 장치로 인한 빠른 CPU의 병목 현상 방지
    - 단순 데이터 전송을 DMA 제어기(DMA Controller)에 위임하여 시스템 전체 처리량(Throughput) 극대화
    

### 2. 동작 과정

1. **명령 하달:** CPU가 DMA 제어기에 전송 정보(I/O 주소, 메모리 시작 주소, 데이터 크기, R/W 여부 등) 전달 및 전송 명령
2. **작업 위임:** CPU는 시스템 버스 제어권을 DMA에 넘기고 본연의 연산 수행
3. **데이터 전송:** DMA 제어기가 시스템 버스를 제어하여 I/O 장치와 메모리 간 데이터 직접 전송
4. **인터럽트 발생:** 전체 데이터 전송 완료 시, DMA 제어기가 CPU에 완료 인터럽트 신호 송신 (CPU는 전송의 시작과 끝에만 관여)

### 3. 버스 사용 방식 (시스템 버스 공유 기술)

CPU와 DMA 제어기가 공용 시스템 버스를 사용하는 방식에 따라 분류

- **사이클 스틸링 (Cycle Stealing):**
    - CPU가 시스템 버스를 사용하지 않는 버스 사이클을 일시적으로 빼앗아 사용
    - 주로 1 Word 단위 전송
    - CPU의 상태 보존 필요 없음. CPU 연산 방해 최소화
    - 예시: 사운드 카드
- **버스트 모드 (Burst Mode):**
    - DMA 제어기가 버스 제어권 획득 시, 데이터 블록 전체 전송 완료 시까지 버스 독점
    - 초고속 데이터 전송 가능
    - 전송 중 CPU의 메모리 접근 불가로 타 작업 지연 발생 가능
    - 예시: 그래픽 카드, 보조기억장치

### 4. 인터럽트 기반 I/O vs DMA 비교

| **구분** | **인터럽트 기반 I/O (Interrupt-driven I/O)** | **DMA (Direct Memory Access)** |
| --- | --- | --- |
| **데이터 전송 주체** | CPU | DMA 제어기 |
| **인터럽트 발생 시점** | 매 데이터(Word/Byte) 전송 완료 시마다 | 전체 데이터 블록 전송 완료 시 1회 |
| **데이터 전송 단위** | Word 또는 Byte | Block |
| **CPU 오버헤드** | 높음 (전송 과정을 CPU가 직접 처리) | 낮음 (명령 하달 및 완료 확인만 수행) |
| **주요 사용처** | 키보드, 마우스 등 소량의 데이터 전송 | 하드 디스크, 그래픽 카드 등 대용량 데이터 전송 |
