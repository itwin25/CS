# 가상 메모리(Virtual Memory) 핵심 요약

## 1. 개요 및 정의

- **정의:** 프로세스 전체가 메모리에 올라가지 않아도 실행 가능하게 하는 **메모리 관리 기법**
- **핵심 목적:** 물리 메모리(RAM) 크기 한계 극복 및 다중 프로그래밍 효율성 향상
- **논리 주소(Logical Address):** CPU가 생성하는 주소, 프로세스별 독립적인 가상 공간
- **물리 주소(Physical Address):** 실제 RAM에 할당된 하드웨어 주소
- **MMU(Memory Management Unit):** 논리 주소를 물리 주소로 변환하는 하드웨어 장치

---

## 2. 주요 메커니즘

### 페이징(Paging)

- 가상 메모리를 고정 크기인 **페이지(Page)** 로, 물리 메모리를 **프레임(Frame)** 으로 분할 관리
- **페이지 테이블(Page Table):** 페이지와 프레임의 매핑 정보를 저장하는 자료구조 (e.g. 1번 페이지는 5번 프레임에 존재)
- **내부 단편화(Internal Fragmentation):** 프로세스 크기가 페이지 배수가 아닐 때 마지막 페이지에 남는 공간

### 요구 페이징(Demand Paging)

- **정의:** 실행에 필요한 페이지만 메모리에 적재하는 방식
- **페이지 폴트(Page Fault):** 접근하려는 페이지가 현재 RAM에 없는 경우 발생하는 인터럽트
- **유효-무효 비트(Valid-Invalid Bit):** 해당 페이지가 메모리에 있는지 여부를 표시

---

## 3. 주소 변환 및 가속 기술

### TLB (Translation Lookaside Buffer)

- **정의:** 페이지 테이블의 일부 내용을 담는 **고속 캐시 메모리**
- **효과:** 주소 변환을 위해 메모리에 두 번 접근(테이블 확인 + 데이터 접근)해야 하는 오버헤드 감소
- **TLB Hit:** TLB에서 즉시 주소 변환 성공 / **TLB Miss:** 페이지 테이블을 직접 참조

### 다단계 페이징(Multi-level Paging)

- **특징:** 페이지 테이블 자체를 다시 페이징하여 계층적으로 관리
- **장점:** 모든 페이지 테이블을 메모리에 상주시키지 않아도 되므로 **메모리 공간 절약**

---

## 4. 페이지 교체 알고리즘 (Page Replacement)

| **알고리즘** | **교체 기준 및 특징** | **비고** |
| --- | --- | --- |
| **FIFO** | 가장 먼저 적재된 페이지 교체 | Belady's Anomaly 발생 가능 |
| **LRU** | 가장 오랫동안 사용되지 않은 페이지 교체 | 시간 국부성 활용, 실제 가장 많이 쓰임 |
| **LFU** | 참조 횟수가 가장 적은 페이지 교체 | 초기 집중 사용된 페이지 문제 발생 가능 |
| **Clock (NUR)** | 참조 비트를 사용하여 시계 방향 탐색 | LRU의 하드웨어적 근사 알고리즘 |

---

## 5. 성능 저하 및 해결 방안

### 스래싱 (Thrashing)

- **현상:** 페이지 폴트가 빈번하여 CPU가 연산보다 **페이지 교체 작업에 더 많은 시간을 쓰는 상태**
- **원인:** 프로세스에게 할당된 프레임 수가 필요량보다 부족할 때 발생
- **해결책:** 워킹 세트(Working Set) 모델 적용, PFF(Page Fault Frequency) 조절

### 단편화 (Fragmentation)

- **내부 단편화:** 고정 크기 할당(페이징) 시 발생
- **외부 단편화:** 가변 크기 할당(세그멘테이션) 시 발생하며, 메모리 압축 등으로 해결
