# 이중 모드 (Dual Mode: User Mode / Kernel Mode)

## 1. 개념

**이중 모드(Dual Mode)** 는 운영체제가 시스템을 보호하기 위해 **CPU 실행 권한을 두 가지 모드로 나누는 구조**이다.

| 모드 | 설명 |
|---|---|
| User Mode | 일반 프로그램이 실행되는 모드 |
| Kernel Mode | 운영체제(OS)가 실행되는 모드 |

즉,

- 일반 프로그램 → **User Mode**
- 운영체제 → **Kernel Mode**

에서 실행된다.

---

## 2. 왜 필요한가?

만약 모든 프로그램이 **운영체제와 동일한 권한**을 가진다면 문제가 발생한다.

예를 들어

- 프로그램이 **메모리를 마음대로 접근**
- **디스크 직접 제어**
- **CPU 제어**
- **다른 프로그램 메모리 접근**

이 경우 다음과 같은 문제가 발생한다.

- 시스템 **충돌(Crash)**
- **보안 문제**
- **데이터 손상**

따라서 운영체제는 시스템을 보호하기 위해

> 일반 프로그램 → **제한된 권한(User Mode)**  
> 운영체제 → **전체 권한(Kernel Mode)**

으로 구분한다.

---

## 3. 두 모드의 권한 차이

| 기능 | User Mode | Kernel Mode |
|---|---|---|
| 일반 프로그램 실행 | 가능 | 가능 |
| 메모리 직접 접근 | 제한 | 가능 |
| 하드웨어 접근 | 불가능 | 가능 |
| 디스크 제어 | 불가능 | 가능 |
| 네트워크 장치 제어 | 불가능 | 가능 |

즉,

> **위험한 작업은 Kernel Mode에서만 가능하다.**

---

## 4. 모드 전환 (Mode Switching)

프로그램이 **운영체제 기능이 필요할 때**는  
User Mode → Kernel Mode로 전환해야 한다.

이때 사용하는 것이 **System Call**이다.

### 예시 동작 흐름
```
User Program 실행
↓
System Call 요청
↓
CPU가 Kernel Mode로 전환
↓
OS가 작업 수행
↓
다시 User Mode로 복귀
```


---

## 5. 예시 카카오톡에서 파일 다운받기

```
카카오톡 실행
↓
User Mode

사진 다운로드 클릭
↓
System Call
↓
Kernel Mode (OS가 네트워크 + 디스크 작업 수행)

작업 완료
↓
User Mode 복귀
↓
카톡이 파일 사용
```


즉,

> **하드웨어 접근은 반드시 OS를 통해서만 가능하다.**

---

## 6. 모드 구분 방법

CPU에는 **Mode Bit**라는 값이 존재한다.

| 값 | 의미 |
|---|---|
| 0 | Kernel Mode |
| 1 | User Mode |

CPU는 이 값을 통해 현재 실행 중인 모드를 판단한다.

---

## 7. 핵심 정리

> 이중 모드는 **시스템 보호를 위해 CPU 실행 권한을 User Mode와 Kernel Mode로 분리하는 구조**이다.

- User Mode → 일반 프로그램 실행
- Kernel Mode → 운영체제 실행
- System Call → 모드 전환
- Kernel Mode에서만 하드웨어 접근 가능
