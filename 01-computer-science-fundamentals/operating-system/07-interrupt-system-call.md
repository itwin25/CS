# 인터럽트와 시스템 콜

## 인터럽트 (Interrupt)

### 정의

CPU가 프로그램을 실행하고 있을 때, **예외 상황이나 급하게 처리해야 할 이벤트가 발생했음을 CPU에게 알리는 신호**

### 주요 작동 순서

1. **인터럽트 발생** : 하드웨어나 소프트웨어가 CPU에 신호를 보냄
2. **현재 상태 저장 :** CPU가 하던 작업의 진행 상황(Context)을 메모리에 안전하게 저장
3. **인터럽트 서비스 루틴(ISR) 실행 :** 발생한 인터럽트의 종류를 파악하고, 그에 맞는 처리 코드를 찾아 실행
4. **상태 복구 및 재개 :** 처리가 끝나면 저장해 둔 원래 작업 상태를 불러와 중단되었던 지점부터 마저 실행

### 인터럽트의 종류

- **하드웨어 인터럽트**
    - 키보드 입력, 마우스 움직임, 타이머, 디스크 읽기 완료 등 물리적인 하드웨어 장치가 발생시키는 신호
- **소프트웨어 인터럽트 (Trap)**
    - 프로그램 실행 중 치명적인 오류(0으로 나누기 등)가 발생하거나, 프로그램이 운영체제에 특정 서비스를 요청할 때 발생

## 시스템 콜 (System Call)

### 정의

응용 프로그램이 운영체제(OS)의 커널(Kernal)이 제공하는 서비스를 사용하기 위해 호출하는 인터페이스(창구)

### 필요한 이유

<aside>
✅

**시스템의 보안과 안정성**

일반 프로그램이 메모리나 하드웨어에 마음대로 접근하면 전체 시스템이 망가질 위험이 있음

</aside>

- **사용자 모드 (User Mode)**
    - 제한된 권한만 가진 상태
    - 일반적인 응용 프로그램 코드가 실행되는 공간
- **커널 모드 (Kernel Mode)**
    - 시스템의 모든 권한을 가진 상태
    - 운영체제의 핵심 기능이 실행되는 공간

응용 프로그램이 파일 읽기, 화면 출력, 네트워크 통신 등 하드웨어 제어가 필요할 때에는 직접 할 수 없음.

대신 시스템 콜을 통해 OS에 대신 수행해달라고 요청하고 이때, 잠시 커널 모드로 권한이 전환되어 작업이 수행됨

## 인터럽트와 시스템 콜의 관계

- 시스템 콜을 호출하는 행위 자체가 내부적으로는 일종의 소프트웨어 인터럽트를 발생시키는 과정

| **구분** | **인터럽트 (Interrupt)** | **시스템 콜 (System Call)** |
| --- | --- | --- |
| **핵심 목적** | CPU의 주의를 환기하여 급한 이벤트를 처리 | 응용 프로그램이 OS의 권한/서비스를 빌려 쓰기 위함 |
| **발생 주체** | 주로 하드웨어 주변 장치 (또는 예외 상황) | 실행 중인 응용 프로그램 (소프트웨어 코드) |
| **동작 방식** | 발생 시 하던 일을 멈추고 커널 모드로 진입 | 호출 시 자발적으로 커널 모드로 진입 |
| **성격** | 수동적 / 비동기적 (언제 일어날지 예측 불가능) | 능동적 / 동기적 (프로그램이 필요할 때 의도적으로 호출) |

---

### 개념정리..

- 악기 연주자들 (하드웨어): CPU, 메모리, 하드디스크, 키보드, 모니터 등

- 악보 (응용 프로그램): 웹 브라우저, 게임, 메신저 등

- 지휘자 (운영체제)
